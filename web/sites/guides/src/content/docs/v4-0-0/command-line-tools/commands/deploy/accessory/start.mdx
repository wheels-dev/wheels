`docker start <service>-<name>` on the accessory's pinned host(s). Fails if the container doesn't exist — use [`accessory boot`](/v4-0-0/command-line-tools/commands/deploy/accessory/boot/) for first-time install.

## Synopsis

``` title="Synopsis"
wheels deploy accessory start <name|all> [--destination=<name>] [--dry-run]
```

## Flags

| Flag | Description |
|------|-------------|
| `<name>` | **Required positional.** Accessory name, or `all` to fan out. |
| `--destination=<name>` | Overlay destination config. |
| `--dry-run` | Print commands without executing. |

## Example

```bash title="Start a named accessory"
wheels deploy accessory start db
```
