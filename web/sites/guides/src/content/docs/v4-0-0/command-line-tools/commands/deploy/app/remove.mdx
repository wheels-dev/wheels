Stop and remove app containers for the given version. Scoped to one version — use [`wheels deploy remove`](/v4-0-0/command-line-tools/commands/deploy/remove/) for a broader teardown.

## Synopsis

``` title="synopsis"
wheels deploy app remove --release=<v> [--role=<name>] [--destination=<name>] [--dry-run]
```

## Flags

| Flag | Description |
|------|-------------|
| `--release=<v>` | **Required.** |
| `--role=<name>` | Limit to one role. |
| `--destination=<name>` | Overlay destination config. |
| `--dry-run` | Print commands without executing. |

## Behavior

For each host in each targeted role, chains:

``` title="illustrative — internal docker commands"
docker stop <service>-<role>-<version> && docker rm <service>-<role>-<version>
```

Fails per-host if the container doesn't exist.

Prefer [`wheels deploy prune`](/v4-0-0/command-line-tools/commands/deploy/prune/) for retention-policy-driven cleanup. `app remove` is surgical — it removes exactly the version you named.

## Example

```bash title="illustrative — requires deploy infrastructure"
wheels deploy app remove --release=abc1234
```
