## Synopsis

```bash
wheels info [options]
```

Prints a one-screen summary of the current Wheels app: CLI version, project path, engine, datasource, `.env` presence, model count, and server status. Intended as a first diagnostic for "does this app look right?"

## Options

| Flag          | Default | Description                               |
| ------------- | ------- | ----------------------------------------- |
| `--json`      | off     | Emit machine-readable JSON instead of a formatted report. |
| `--quiet`     | off     | Suppress terminal colors.                 |

## Exit codes

| Code | Meaning                                   |
| ---- | ----------------------------------------- |
| `0`  | Report printed successfully.              |
| `1`  | Not inside a Wheels app directory.        |

## Examples

Show the current app's status:

```bash {test:cli cmd="wheels info" asserts-output="Wheels CLI"}
wheels info
```

Pipe the JSON form into another tool:

```bash title="your shell"
wheels info --json | jq '.models'
```
