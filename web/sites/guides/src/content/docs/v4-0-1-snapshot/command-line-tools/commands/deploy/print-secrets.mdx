Print the fully resolved `.kamal/secrets` file as `KEY=VALUE` lines. All `$(...)` subshells expand.

This is the **canonical CLI form** for the print operation. The Kamal-style nested `wheels deploy secrets print` is also accepted by the deploy dispatcher, but the CLI runtime registers `secrets` as a top-level command (for its own credential store) and intercepts the three-token form before deploy can see it. The flat alias sidesteps the collision — see [issue #2697](https://github.com/wheels-dev/wheels/issues/2697).

<Aside type="caution">
Output is plaintext secrets. Never paste this into logs, tickets, or shared terminals. Run in private.
</Aside>

## Synopsis

``` title="synopsis"
wheels deploy print-secrets [--destination=<name>]
```

## Flags

| Flag | Description |
|------|-------------|
| `--destination=<name>` | Overlay `.kamal/secrets.<name>` on top of `.kamal/secrets`. |

## Behavior

Uses the internal `SecretResolver` — the same resolver the deploy flow uses. Every `$(...)` subshell runs through `bash`. Missing secrets referenced in `deploy.yml` do not appear in the output; they'd fail the deploy separately. Operates against the current directory.

Useful to audit what the deploy flow will see before running it.

## Example

```bash title="illustrative"
wheels deploy print-secrets
wheels deploy print-secrets --destination=production
```
