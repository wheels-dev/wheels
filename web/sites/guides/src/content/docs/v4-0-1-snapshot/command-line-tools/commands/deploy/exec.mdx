Run a command on every host. Filter to one host with `--host=`.

This is the **canonical CLI form** for the exec operation. The Kamal-style nested `wheels deploy server exec` is also accepted by the deploy dispatcher, but the CLI runtime registers `server` as a top-level command (for Lucee server lifecycle) and intercepts the three-token form before deploy can see it. The flat alias sidesteps the collision — see [issue #2677](https://github.com/wheels-dev/wheels/issues/2677).

## Synopsis

``` title="synopsis"
wheels deploy exec <command...> [--host=<host>] [--destination=<name>] [--dry-run]
```

## Flags

| Flag | Description |
|------|-------------|
| `--host=<host>` | Limit to one host. Must match a host declared in `deploy.yml` — throws `DeployServerCli.UnknownHost` otherwise. |
| `--destination=<name>` | Overlay destination config. |
| `--dry-run` | Print the command without executing. |

## Behavior

Joins all positional tokens after `exec` into a single command string, SSHes in parallel (or to one host), and runs it verbatim. Output is prefixed by host. Throws `DeployServerCli.MissingCommand` if no command is supplied.

Useful for smoke-testing connectivity, reading environment details, or running one-off diagnostic commands without writing a hook.

## Example

```bash title="illustrative — requires live SSH hosts"
wheels deploy exec "uptime"
wheels deploy exec "docker ps" --host=192.0.2.10
```
