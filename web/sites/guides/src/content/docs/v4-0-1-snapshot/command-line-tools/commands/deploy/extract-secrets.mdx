Extract one key's value from a `KEY=VALUE` block. Useful inside hooks or wrapper scripts that pipe from `wheels deploy fetch-secrets`.

This is the **canonical CLI form** for the extract operation. The Kamal-style nested `wheels deploy secrets extract` is also accepted by the deploy dispatcher, but the CLI runtime registers `secrets` as a top-level command (for its own credential store) and intercepts the three-token form before deploy can see it. The flat alias sidesteps the collision — see [issue #2697](https://github.com/wheels-dev/wheels/issues/2697).

## Synopsis

``` title="synopsis"
wheels deploy extract-secrets <KEY> --from=<block>
```

## Flags

| Flag | Description |
|------|-------------|
| `<KEY>` | Positional. Key to extract. |
| `--from=<block>` | The `KEY=VALUE` newline-separated text to extract from. |

## Behavior

Scans `--from` line-by-line and returns the value for the first line whose key equals the positional `<KEY>`. Returns an empty string if the key isn't present or if no positional key is supplied.

## Example

```bash title="illustrative — shell variables not supported by harness"
block=$(wheels deploy fetch-secrets --adapter=op --from=op://Prod/App A B)
a=$(wheels deploy extract-secrets A --from="$block")
```
