Secret-store integration. Five built-in adapters (1Password, Bitwarden, AWS Secrets Manager, LastPass, Doppler), exposed behind three verbs.

:::caution[Prefer the flat aliases at the shell]
The nested `wheels deploy secrets <verb>` forms are accepted by the deploy dispatcher but do not work when invoked through the wheels CLI — the runtime intercepts `secrets` for its own credential store before deploy sees it. Use the flat aliases instead. See [issue #2697](https://github.com/wheels-dev/wheels/issues/2697).
:::

## Verbs

| Verb | Flat alias (canonical CLI form) | Purpose |
|------|---------------------------------|---------|
| [`fetch`](/v4-0-1-snapshot/command-line-tools/commands/deploy/secrets/fetch/) | [`wheels deploy fetch-secrets`](/v4-0-1-snapshot/command-line-tools/commands/deploy/fetch-secrets/) | Pull keys from a named adapter. |
| [`extract`](/v4-0-1-snapshot/command-line-tools/commands/deploy/secrets/extract/) | [`wheels deploy extract-secrets`](/v4-0-1-snapshot/command-line-tools/commands/deploy/extract-secrets/) | Extract one key from a `KEY=VALUE` block. |
| [`print`](/v4-0-1-snapshot/command-line-tools/commands/deploy/secrets/print/) | [`wheels deploy print-secrets`](/v4-0-1-snapshot/command-line-tools/commands/deploy/print-secrets/) | Print the resolved `.kamal/secrets`. |

## Adapter names

Each adapter is registered under one or two aliases matching Kamal's CLI conventions.

| Adapter | Aliases |
|---------|---------|
| 1Password | `op`, `1password` |
| Bitwarden | `bw`, `bitwarden` |
| AWS Secrets Manager | `aws` |
| LastPass | `lpass`, `lastpass` |
| Doppler | `doppler` |

Adapters shell out to the corresponding CLI tool on the control machine. See [Secrets](/v4-0-1-snapshot/deployment/secrets/) for setup.
