- `Seeder.runSeeds()` no longer reports success and commits when individual `seedOnce()` entries failed validation. Failed entries are now counted (`totalFailed` in the result struct and the `/wheels/cli` dbSeed envelope), named in the failure message, and the entire run is rolled back — `seedOnce()` is idempotent, so a corrected rerun re-applies everything, whereas a silently half-committed run looked identical to a fully-applied one in CI and `wheels seed` output. Because the dev-UI bridge already maps `success=false` to `Wheels.Cli.CommandFailed`, `wheels seed` now exits non-zero when any entry fails (#2973)
