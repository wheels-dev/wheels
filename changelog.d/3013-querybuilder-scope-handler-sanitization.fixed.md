- `QueryBuilder.onMissingMethod` now sanitizes dynamic scope handler arguments via `$sanitizeScopeHandlerArgs`, matching the existing behavior of the model-root and `ScopeChain` invocation paths (#3013). Previously, `model.where(...).byHandlerScope(input)` delegated the raw `missingMethodArguments` to the handler, while `model.byHandlerScope(input)` and `model.someScope().byHandlerScope(input)` ran through the sanitizer — so the same scope produced different `where` specs depending on how it was reached.
