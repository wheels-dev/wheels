- The app-template reload redirect no longer strips the environment-switch params, so `?reload=<env>&password=...` actually switches environments instead of being a silent no-op. `$buildRedirectUrl()` now preserves `reload`+`password` through the post-`applicationStop()` redirect when the reload value names an environment (a non-boolean `url.reload`), while still stripping them for `?reload=true`; the `onRequestStart()` reload gate gains a loop-break that skips `applicationStop()` once `application.wheels.environment` already equals `url.reload`, so the now-preserved params can't loop. Applied identically to all four `Application.cfc` lineage copies (`public/`, the `wheels new` template, and the `starter-app`/`tweet` examples). Trade-off: `?reload=<current-env>` becomes a no-op — use `?reload=true` to force a same-environment restart (#3030)
