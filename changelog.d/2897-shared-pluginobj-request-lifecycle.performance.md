- Request-lifecycle mixin injection (`onDIcomplete` on controllers, model objects and the dispatcher, plus `$runOnRequestStart`) now reuses the application-cached `Plugins` instance via the new `$pluginObj()` helper instead of constructing a throwaway `wheels.Plugins` — and paying its 4,000-line `wheels.Global` parent pseudo-constructor — per request and per materialized model row. `$initializeMixins` keeps its scratch state (`appKey`/`metaData`/`className`) strictly local-scoped so the shared instance is safe across concurrent requests, and the `$runOnRequestStart` construction is hoisted inside the mixins-nonempty guard so mixin-free apps skip it entirely (#2897, Stage 3)
