- `wheels deploy config --destination=<dest>` now applies the destination overlay (`deploy.<dest>.yml`) before dumping the resolved YAML, matching `deploy`/`details`/`audit`. Previously the destination was silently ignored and the un-overlaid base config was printed, breaking the `kamal config` diff step in the Kamal migration checklist ([#3085](https://github.com/wheels-dev/wheels/issues/3085))
