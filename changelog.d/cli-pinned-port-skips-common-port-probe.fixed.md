- CLI server detection no longer falls back to probing the common ports (8080/60000/3000/8500) when the project pins its own port in `lucee.json` or `.env` — a pinned-but-closed port now resolves to "no running Wheels server" instead of silently attaching to whatever foreign process answers on 8080 (cross-project hazard from #2878; surfaced by the #3170 docs-verify refusal blocks inverting under a stray local listener). The docs-verify harness also rewrites every `wheels new` fixture onto closed ephemeral ports so the documented refusal blocks stay deterministic on contributor machines.
