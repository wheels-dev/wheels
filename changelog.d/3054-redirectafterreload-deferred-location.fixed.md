- URL environment switches into `production`/`maintenance` (`?reload=<env>&password=...`) no longer 500 and silently revert: the `redirectAfterReload` block in `wheels/events/onapplicationstart.cfc` called a bare `$location()` that does not resolve in that mixin-free component, and even a resolvable redirect there aborts `onApplicationStart` mid-flight, making the engine discard the half-started application. The redirect is now deferred to `EventMethods.$runOnRequestStart`, which runs in the same request after the new application — including the switched environment — has been persisted ([#3054](https://github.com/wheels-dev/wheels/issues/3054))
