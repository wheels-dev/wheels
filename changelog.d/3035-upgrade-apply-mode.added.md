- `wheels upgrade` gains an apply mode: bare `wheels upgrade` now swaps the app's vendored `vendor/wheels/` for the framework bundled inside the installed CLI, backing the old copy up to `vendor/wheels.bak-<timestamp>` via an atomic rename first. Safety rails refuse outside a Wheels app, when source and target resolve to the same directory, and when `--to=<version>` does not match the bundled framework version; `--dry-run` previews the from → to swap and backup path without writing, `--nobackup` opts out of the backup, and `--skip-check` skips the preflight breaking-change scan. `wheels upgrade check` keeps its read-only behavior (including `--strict` and `--format=json`) unchanged. The swap logic lives in the pure, unit-testable `cli/lucli/services/FrameworkSwap.cfc` (`plan()` / `apply()`) (#3035)
