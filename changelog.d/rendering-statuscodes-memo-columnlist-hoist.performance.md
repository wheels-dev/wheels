- `$getStatusCodes()` no longer rebuilds its 63-entry HTTP status map on every render: the map is memoized in the application scope together with a deterministic reverse (text-to-code) lookup, so `$returnStatusCode("Not Found")` is a struct-key hit instead of a `StructFindValue` scan over a freshly rebuilt struct, and a duplicated status text (e.g. "Unassigned" at 427/430/509) now always resolves to the lowest matching code (#2961)
- `$includeFile()` tokenizes the query column list once per partial render instead of re-running `ListToArray(query.columnList)` on every row of both per-row loops (#2961)
