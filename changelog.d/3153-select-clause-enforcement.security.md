- `findAll(select=)` now **enforces** the SEC-21 raw-SQL policy that the 4.x deprecation window only warned about: select items containing a statement separator (`;`), comment marker (`--` / `/*`), or a parenthesized subquery throw `Wheels.InvalidSelectClause` (mirroring the existing `Wheels.InvalidOrderClause` / `Wheels.InvalidGroupClause` throws on `ORDER BY` / `GROUP BY`) instead of passing through unvalidated. Plain columns, `table.column`, single-argument functions, and `expr AS alias` items still pass; prefer a calculated property declared with `property()` in `config()` for reusable computed columns. A new `allowRawSelect=true` argument on `findAll` / `findOne` opts a deliberately audited, request-input-free expression out of the check — the single greppable escape hatch every peer ORM ships alongside this kind of enforcement. The warn-only `$warnOnUnvalidatedSelectItem` branch is removed ([#3153](https://github.com/wheels-dev/wheels/issues/3153))
