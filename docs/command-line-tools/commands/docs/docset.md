# Generate Docset

Generate a Dash/Zeal compatible docset from CFWheels API documentation.

## Usage

```bash
wheels docs docset [options]
```

## Options

| Option | Description | Default |
|--------|-------------|---------|
| `--output` | Output directory for the generated docset | Current directory |
| `--name` | Name for the docset | "CFWheels" |
| `--author` | Author name for the docset | "CFWheels Team" |
| `--verbose` | Show detailed output during generation | false |

## Examples

### Basic usage
```bash
# Generate docset in current directory
wheels docs docset

# Generate docset in specific directory
wheels docs docset --output ~/Desktop

# Generate with custom name and verbose output
wheels docs docset --name "CFWheels-3.0" --verbose --author "My Team"
```

## What is a Docset?

A docset is an offline documentation format used by documentation browsers like:

- **Dash** (macOS) - Premium documentation browser
- **Zeal** (Linux/Windows) - Free open-source alternative  
- **VSCode Extensions** - Integration with popular editors

## Installation & Setup

### 1. Generate the Docset
```bash
# In your CFWheels project
wheels docs docset --output ~/Downloads
```

This creates: `~/Downloads/CFWheels.docset`

### 2. Install Documentation Browser

**macOS - Dash:**
- Install from App Store or https://kapeli.com/dash
- Go to Preferences → Docsets → "+" → Add Local Docset
- Select the `.docset` file

**Windows/Linux - Zeal:**
- Download from https://zealdocs.org/
- Go to Tools → Docsets → Add → Browse for docset
- Select the `.docset` file

### 3. VSCode Integration (Optional)

**macOS:**
```bash
# Install Dash extension
code --install-extension deerawan.vscode-dash
```

**Windows/Linux:**
```bash
# Install Zeal extension  
code --install-extension deerawan.vscode-zeal
```

## Developer Workflow

Once installed, you can instantly search CFWheels functions:

1. **In VSCode:** Select function name → `Cmd+Shift+D` (Mac) or `Ctrl+Shift+D` (Windows)
2. **In Dash/Zeal:** Use global hotkey or search directly in app
3. **Quick lookup:** Type "findAll" → see parameters instantly

## Example Output

The command generates:

```
CFWheels.docset/
├── Contents/
│   ├── Info.plist           # Metadata
│   └── Resources/
│       ├── docSet.dsidx     # SQLite search index
│       └── Documents/       # HTML documentation
│           ├── index.html
│           ├── section_*.html
│           ├── function_*.html
│           └── css/style.css
```

## Requirements

- Must be run in a CFWheels project directory
- Requires API documentation JSON (generated by visiting `/wheels/api?format=json`)
- SQLite support for search index creation

## Benefits

- **Offline access** - No server needed
- **IDE integration** - Search from your editor
- **Instant search** - Faster than web browsing
- **Professional** - Same format as Rails, Laravel, etc.

## Troubleshooting

### "No API documentation JSON file found"
```bash
# Start your CFWheels application
server start

# Visit in browser to generate API JSON
http://localhost:8080/wheels/api?format=json

# Then run docset command
wheels docs docset
```

### Search index creation fails
The command will still generate a functional docset without SQLite search. The docset will work in Dash/Zeal but search may be limited.

### Docset doesn't appear in Dash/Zeal
1. Verify the `.docset` extension is present
2. Check that `Info.plist` exists in `Contents/` directory  
3. Try restarting Dash/Zeal after adding the docset
